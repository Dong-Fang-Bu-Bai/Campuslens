## 🎯 一、问题背景与目标

### 1.1 问题描述

当前 CampusLens 图像检索系统面临以下挑战：

| 问题类型 | 具体表现 | 影响 |
|---------|---------|------|
| **图像噪声** | 输入图像可能包含高斯噪声、模糊、压缩失真等 | 特征提取不稳定，检索准确率下降 |
| **标签不准确** | 测试时获得的地标标签可能存在错误 | 影响检索结果排序和召回率 |
| **域偏移** | 测试数据与训练数据分布不一致 | 模型泛化能力受限 |

### 1.2 目标

集成 **SAR (Sharpness-Aware and Reliable entropy minimization)** 算法，实现：

- ✅ 提升噪声图像的检索准确率
- ✅ 利用标签信息增强自适应能力
- ✅ 在线适应测试数据分布变化
- ✅ 保持系统低延迟响应

## 🏛️ 二、SAR 算法原理

### 2.1 算法概述

**SAR**（ICLR 2023 Oral）是一种测试时自适应算法，核心思想：

1. **可靠样本过滤**：通过熵阈值过滤预测不确定的样本
2. **Sharpness-Aware 优化**：使用 SAM 优化器寻找更平坦的极小值
3. **模型恢复机制**：EMA 监测防止过拟合到测试数据

### 2.2 算法流程图

```mermaid
flowchart TD
    A[输入图像 x] --> B[第一次前向传播]
    B --> C{计算 softmax 熵 H}
    C --> D{过滤可靠样本 H 小于 margin}
    D -->|是| E[计算损失 loss = mean filtered_entropy]
    D -->|否| F[跳过本次更新]
    E --> G[第一次反向传播]
    G --> H[SAM 第一步: 参数扰动]
    H --> I[第二次前向传播]
    I --> J{再次计算熵 H2}
    J --> K{二次过滤 H2 小于 margin}
    K -->|是| L[更新 EMA]
    K -->|否| M[跳过更新]
    L --> N[SAM 第二步: 参数更新]
    N --> O{EMA 小于 0.2?}
    O -->|是| P[重置模型]
    O -->|否| Q[返回输出]
    F --> Q
    M --> Q
    P --> Q
```

### 2.3 核心公式

| 公式 | 描述 |
|------|------|
| `H(p) = -Σ p_i * log(p_i)` | Softmax 熵计算 |
| `filter_ids = {i \| H(p_i) < margin_e0}` | 可靠样本过滤 |
| `EMA_new = 0.9 * EMA_old + 0.1 * loss` | 指数移动平均更新 |
| `if EMA < reset_constant → reset` | 模型恢复条件 |

## 🔧 三、标签增强的 SAR 方案

### 3.1 设计思路

将标签信息融入 SAR 自适应流程，形成**标签引导的自适应策略**：

```mermaid
flowchart LR
    A[图像 x] --> B[标签 y]
    B --> C{标签置信度}
    C -->|大于 0.8| D[强标签引导]
    C -->|0.5-0.8| E[混合引导]
    C -->|小于 0.5| F[弱标签引导]
    D --> G[标签熵与原始熵融合]
    E --> G
    F --> G
    G --> H[SAR 自适应]
    H --> I[输出特征]
```

### 3.2 融合熵计算

**融合熵计算公式：**

```
H_fused = λ_conf * H_label + (1 - λ_conf) * H_original
```

其中：
- `H_label = -log(p(label))`：标签类别的负对数概率
- `H_original = softmax_entropy(outputs)`：原始熵
- `λ_conf`：标签置信度 [0, 1]

### 3.3 标签置信度处理策略

| 置信度范围 | 策略 | 标签权重 |
|-----------|------|---------|
| `[0.8, 1.0]` | 高置信度，完全信任标签 | 0.8 ~ 1.0 |
| `[0.5, 0.8)` | 中等置信度，混合策略 | 0.5 ~ 0.8 |
| `[0.3, 0.5)` | 低置信度，弱引导 | 0.2 ~ 0.5 |
| `[0, 0.3)` | 极低置信度，忽略标签 | 0 |

## 👥 四、用户交互驱动的自适应场景

### 4.1 场景概述

CampusLens 系统支持两种用户交互模式，完美契合 SAR 算法的应用场景：

| 交互模式 | 标签情况 | SAR 策略 | 适用场景 |
|---------|---------|---------|---------|
| **用户上传图片** | 无标签 | 纯 SAR 熵最小化自适应 | 日常检索使用 |
| **用户主动反馈** | 带标签 | 标签引导的 SAR 自适应 | 纠错、补充标注 |

### 4.2 场景一：用户上传图片（无标签自适应）

```mermaid
flowchart TD
    A[用户上传图片] --> B[DINOv2 特征提取]
    B --> C[SAR 无标签自适应]
    C --> D[熵最小化更新模型]
    D --> E[FAISS 检索]
    E --> F[返回 Top-K 结果]
    
    subgraph SAR自适应过程
        D --> D1[计算 softmax 熵]
        D1 --> D2[过滤可靠样本]
        D2 --> D3[SAM 参数更新]
        D3 --> D4[EMA 监测]
        D4 --> D5{需要恢复?}
        D5 -->|是| D6[重置模型]
        D5 -->|否| D7[继续]
        D6 --> D7
    end
```

**核心特点**：
- ✅ 无需人工标注，自动适应数据分布
- ✅ 通过熵最小化提升预测置信度
- ✅ 可靠样本过滤避免噪声干扰
- ✅ 模型恢复机制防止过拟合

### 4.3 场景二：用户主动反馈（带标签自适应）

```mermaid
flowchart TD
    A[用户反馈: 图片 + 标签] --> B[DINOv2 特征提取]
    B --> C[标签置信度评估]
    C --> D{置信度等级}
    D -->|高置信度 >= 0.8| E[强标签引导]
    D -->|中置信度 0.5-0.8| F[混合引导]
    D -->|低置信度 < 0.5| G[弱标签引导]
    E --> H[SAR 标签引导自适应]
    F --> H
    G --> H
    H --> I[更新模型参数]
    I --> J[可选: 更新 FAISS 索引]
    J --> K[返回确认信息]
    
    subgraph 标签引导自适应
        I --> I1[计算融合熵]
        I1 --> I2[H_fused = λ*H_label + 1-λ*H_original]
        I2 --> I3[SAM 参数更新]
        I3 --> I4[EMA 监测]
    end
```

**核心特点**：
- ✅ 利用用户反馈的标签信息
- ✅ 置信度评估避免错误标签干扰
- ✅ 可选更新 FAISS 索引（新增样本）
- ✅ 持续学习，模型能力不断提升

### 4.4 标签置信度来源

| 来源 | 置信度范围 | 说明 |
|------|-----------|------|
| **用户明确确认** | 0.9 ~ 1.0 | 用户主动选择正确地标 |
| **用户纠错反馈** | 0.8 ~ 0.95 | 用户纠正错误结果 |
| **用户模糊反馈** | 0.5 ~ 0.8 | 用户选择"可能正确" |
| **系统推断标签** | 0.3 ~ 0.6 | 基于检索结果推断 |
| **低质量反馈** | < 0.3 | 忽略，使用纯 SAR |

### 4.7 持续学习策略

| 策略 | 触发条件 | 操作 |
|------|---------|------|
| **模型微调** | 每次用户交互 | SAR 自适应更新归一化层参数 |
| **索引更新** | 用户反馈 + 高置信度 | 将新样本加入 FAISS 索引 |
| **统计更新** | 索引更新时 | 重新计算马氏距离统计量 |
| **模型恢复** | EMA < 0.2 | 重置模型到初始状态 |
| **定期保存** | 每 N 次交互 | 保存模型快照 |

|------|---------|-----------|------|

| **噪声图像准确率** | 70% | 85% | +15% |

### 4.9 风险与应对

| 风险 | 描述 | 应对策略 |
|------|------|---------|
| **错误反馈** | 用户可能提供错误标签 | 置信度评估 + 熵过滤 |
| **概念漂移** | 用户偏好随时间变化 | 模型恢复机制 + 定期重置 |
| **过拟合** | 模型适应到特定用户 | 限制自适应步数 + EMA 监测 |
| **冷启动** | 新用户无历史数据 | 使用预训练模型 + 渐进适应 |

## ⚙️ 六、算法实现细节

### 6.1 参数更新策略

```mermaid
flowchart TD
    A[模型参数] --> B{参数类型}
    B -->|归一化层| C[可更新]
    B -->|Layer4| D[冻结]
    B -->|其他层| D
    C --> E[SAM优化]
    D --> F[保持不变]
```

**参数选择逻辑**（基于 `sar.py` 源码）：

```python
# 可更新参数
if isinstance(m, (nn.BatchNorm2d, nn.LayerNorm, nn.GroupNorm)):
    for np, p in m.named_parameters():
        if np in ['weight', 'bias']:
            params.append(p)

# 冻结参数
# - layer4 (ResNet)
# - blocks.9-11 (ViT-Base)
# - 最后一层 norm
```

### 6.2 SAM 优化器流程

```mermaid
sequenceDiagram
    participant O as Optimizer
    participant M as Model
    participant G as Gradient
    
    Note over O,M: 第一次梯度
    O->>M: zero_grad()
    M->>M: forward(x)
    M->>G: compute_loss()
    G->>M: backward()
    O->>O: first_step
    
    Note over O,M: 第二次梯度
    M->>M: forward(x)
    M->>G: compute_loss()
    G->>M: backward()
    O->>O: second_step
```

### 6.3 模型恢复机制

