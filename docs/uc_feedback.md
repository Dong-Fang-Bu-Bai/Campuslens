| 项目 | 内容 |
|------|------|
| 用例编号 | UC-M5-01 |
| 用例名称 | 用户反馈纠错 |
| 负责人 | 庄子杰 |

| 创建日期 | 2026-05-24 |
| 版本 | v1.0 |

## 参与者

- **主要参与者**：普通用户（已上传图片并获取检索结果）
- **系统**：CampusLens 后端服务

## 前置条件

1. 用户已完成图片上传，并获取 Top-5 地标检索结果。
2. 系统中存在 `landmark` 地标数据。
3. 检索记录已保存至 `search_records` 表。

## 后置条件

### 成功保证

- 系统保存用户反馈记录至 `user_feedback` 表。
- 反馈状态置为 `pending`，等待后续审核。

### 最小保证

- 若保存失败，前端展示错误提示，用户可重新提交。

## 基本流

1. **用户查看检索结果**：系统在结果页展示 Top-5 地标列表，每个结果包含地标名称、相似度得分和代表图片。每个结果行右侧提供"纠错"按钮。
2. **用户点击纠错**：用户对某个检索结果不满意时，点击该结果对应的"纠错"按钮。系统弹出纠错表单，包含以下字段：
   - 当前预测地标名称（只读展示）
   - 反馈类型选择：识别正确 / 识别错误 / 不确定
   - 正确地标选择器（当反馈类型为"识别错误"时必填）
   - 补充说明（选填，最多 500 字）
3. **用户选择正确地标**：当用户选择"识别错误"反馈类型后，系统展示地标下拉列表，列出全部可用地标供用户选择。用户从中选取正确的地标。
4. **用户提交反馈**：用户点击"提交"按钮，系统校验表单完整性（错误类型必须选择正确地标），调用 `POST /api/feedback` 提交反馈数据。
5. **系统保存反馈**：后端接收请求，将反馈数据写入 `user_feedback` 表，关联 `search_records` 记录，返回保存成功的反馈 ID。
6. **前端提示完成**：系统关闭纠错弹窗，展示"反馈已提交，感谢您的帮助"提示信息。

## 备选流

### A1 用户选择"识别正确"

在步骤 3 处，用户选择"识别正确"反馈类型：
- 系统将 `predicted_landmark_id` 与 `confirmed_landmark_id` 设为相同值。
- 正确地标选择器隐藏，无需用户额外操作。
- 继续执行步骤 4。

### A2 用户选择"不确定"

在步骤 3 处，用户选择"不确定"反馈类型：
- `confirmed_landmark_id` 允许为空。
- 正确地标选择器隐藏。
- 继续执行步骤 4。

### A3 表单校验失败

在步骤 4 处，用户提交时未填写必填字段：
- 系统高亮未填写的必填字段，提示"请选择正确地标"。
- 用户补充后重新提交，回到步骤 4。

## 异常流

### E1 网络请求失败

在步骤 5 处，网络异常导致请求未到达后端或响应超时：
- 前端展示"网络异常，请稍后重试"提示。
- 用户可重新点击提交按钮，重新发起请求。

### E2 服务端处理异常

在步骤 5 处，后端处理过程中发生异常（如数据库连接失败）：
- 后端返回 HTTP 500 错误。
- 前端展示"系统繁忙，请稍后重试"提示。

## 业务规则

| 编号 | 规则描述 |
|------|----------|
| BR1 | 每条检索记录可对应多条反馈记录。 |
| BR2 | 反馈类型为 `wrong` 时，`confirmed_landmark_id` 为必填。 |
| BR3 | 反馈类型为 `correct` 时，`confirmed_landmark_id` 默认等于 `predicted_landmark_id`。 |
| BR4 | 反馈类型为 `uncertain` 时，`confirmed_landmark_id` 允许为空。 |
| BR5 | 反馈提交后状态默认为 `pending`，后续由管理员审核变更为 `reviewed` 或 `adopted`。 |

## 涉及接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/feedback` | POST | 提交用户反馈 |

## 涉及数据表

| 表名 | 用途 |
|------|------|
| `search_records` | 检索记录 |
| `user_feedback` | 纠错反馈 |
| `landmark` | 地标信息（查询正确地标列表） |
