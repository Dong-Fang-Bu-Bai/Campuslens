# 初始阶段 M1、M2、M4、M5 完成记录

日期：2026-05-20

本记录用于补充第一周初始阶段交付物，说明 M1、M2、M4、M5 在仓库中的准备情况和第二周衔接口径。

## M1 图片上传与地标检索主流程

已在 `backend/` 建立 Spring Boot 最小工程，提供健康检查、地标列表、地标详情、图片上传检索和反馈提交接口。图片上传接口当前完成基础校验、上传文件保存和 Top-5 演示结果返回，能够支撑前端原型联调和接口样例验证。

当前版本未直接调用算法服务，`SearchService` 保留了接入边界。第二周进入细化阶段后，应将演示结果替换为调用 `algorithm` 的 `POST /api/v1/search`，并补齐 `searchRecordId`、上传图路径和 Top-5 结果快照的数据口径。后端 MySQL 持久化暂不作为第一周实现内容，当前仅完成表结构、种子脚本和接口字段准备。

## M2 地标图像库与元数据管理

首批 L01-L10 地标已经统一编号、名称、英文名、类型、位置描述、简介和地图标注坐标。数据库种子脚本已补充 `description`、`map_x`、`map_y` 和 `cover_image_url`，与前端展示字段和接口契约保持一致。

数据集目录继续保留 `.gitkeep`，原图不直接提交。第一周重点是明确每个地标不少于 20 张样本图片、代表图选择和命名规则；样本图片由成员按 `docs/04_dataset_rules.md` 手动采集、整理并上传到课程指定位置或本地数据目录，第二周再用于算法索引构建和检索验证。

## M4 检索结果展示与地图导览

已在 `frontend/` 建立 Vue 3 + Vite 最小工程，原型包含图片上传区、Top-5 结果卡片、地标详情面板、反馈入口和静态地图标注。页面优先调用后端接口，后端未启动时使用内置演示数据，便于单独展示原型。

静态地图素材归档在 `docs/assets/campus-map.png`，前端运行素材位于 `frontend/public/campus-map.png`。地图标注采用百分比坐标，后续若更换底图，需要同步调整 L01-L10 的 `mapX` 和 `mapY`。

## M5 用户反馈纠错与检索记录统计

反馈接口和前端表单已支持 `correct`、`wrong`、`uncertain` 三类反馈。错误反馈需要提供正确地标，正确反馈可将预测地标和确认地标保持一致，不确定反馈允许不指定确认地标。

当前版本反馈记录暂存为接口响应，不做 MySQL 持久化。第四周可扩展为后台反馈列表、审核状态、采纳记录和统计图表；第二周只需保证结果页可进入反馈入口，并能形成 `searchRecordId` 与反馈数据的衔接。

## 后续衔接

第二周优先打通“上传图片 -> 调用算法服务 -> 返回 Top-5 地标 -> 前端展示结果”的主流程。M2 继续提供基础地标和手动整理的样本图片，M4 在真实结果上完善展示细节，M5 在主流程稳定后补充反馈记录的数据口径和后续持久化方案。
