## 目录结构

```text

    L10_hotel/
```

仓库中可以先保留目录说明，不建议直接提交大量原图。样本图片可放在本地、网盘或压缩包中，交付前再按课程要求整理。

## 图片命名

格式：

```text
<landmarkCode>_<angle>_<light>_<index>.jpg
```

示例：

```text
L01_front_day_001.jpg
L01_side_cloudy_002.jpg
L04_far_day_001.jpg
```

推荐取值：

| 字段 | 说明 | 示例 |
| --- | --- | --- |
| `angle` | 拍摄角度 | `front`、`side`、`far`、`near`、`entrance` |
| `light` | 光照条件 | `day`、`cloudy`、`evening`、`backlight` |
| `index` | 三位序号 | `001`、`002` |

## 采集要求

- 每个地标不少于 20 张图片。
- 覆盖正面、侧面、远景、近景和典型入口或标志物。
- 尽量覆盖不同光照和距离。
- 删除严重模糊、主体过小、遮挡过多、重复度过高的图片。
- 每个地标至少选择 1 张代表图用于前端展示。

## 元数据

每个地标至少维护：

- 地标编号。
- 中文名称。
- 英文或拼音名称。
- 类型。
- 位置描述。
- 简介。
- 代表图片路径。
- 平面图静态标注位置。

### 图像库管理

1. **本地存储**：图像库存储在本地 `datasets/landmarks/` 目录下，每个地标对应一个子目录
2. **版本控制**：图像文件不提交到 Git，通过 `.gitignore` 配置忽略，避免仓库体积过大
3. **文件格式**：统一使用 `.jpg` 格式，便于算法服务处理
4. **代表图标识**：每个地标选择 1 张最具代表性的图片作为封面图，在数据库 `landmark` 表的 `cover_image_url` 字段中记录

## 样本质量风险说明

### 已识别风险

| 风险类型 | 风险描述 | 影响范围 | 严重程度 | 关联模块 |
| --- | --- | --- | --- | --- |
| 命名规范不一致 | 部分图片使用非规范角度值（如 `back`） | 算法服务解析、图片分类 | 中 | M2、M3 |
| 图片质量参差不齐 | 存在模糊、遮挡、主体过小的图片 | 特征提取准确性 | 中 | M3 |
| 光照条件覆盖不均 | 部分地标白天图片较多，阴天/傍晚图片较少 | 检索鲁棒性 | 低 | M3 |
| 角度覆盖不均 | 部分地标缺少远景或近景图片 | 多角度识别能力 | 低 | M3 |
| 重复图片 | 存在角度、构图高度相似的重复图片 | 存储空间、检索效率 | 低 | M2、M3 |
| 代表图未确认 | 部分地标尚未确定代表图 | 前端展示、地图标注 | 中 | M2、M4 |

### 风险应对预案

1. **命名规范问题**：使用 PowerShell 脚本批量检查和修复命名不符合规范的文件
2. **图片质量问题**：建立图片质量评估标准（清晰度、主体比例、遮挡程度），对不符合标准的图片进行二次采集
3. **角度/光照覆盖不足**：制定补充采集计划，确保每个地标覆盖至少 3 种角度和 2 种光照条件
4. **重复图片**：使用图像相似度算法（如感知哈希）检测并删除重复图片

### 质量验收标准

| 指标 | 验收标准 |
| --- | --- |
| 图片数量 | 每个地标不少于 20 张 |
| 命名规范 | 100% 符合 `Lxx_angle_light_00x.jpg` 格式 |
| 角度覆盖 | 每个地标至少覆盖正面、侧面、远景、近景 |
| 光照覆盖 | 每个地标至少覆盖白天、阴天两种光照 |
| 图片质量 | 无严重模糊、主体清晰、遮挡面积 < 30% |
| 重复率 | 重复图片比例 < 5% |
