# 图片采集与数据集规范

## 目录结构

```text
datasets/
  landmarks/
    L01_library/
    L02_academic_auditorium/
    L03_wenyong_square/
    L04_boxue_bridge/
    L05_qin_lake_huxin_island/
    L06_stadium/
    L07_natatorium/
    L08_first_dining_hall/
    L09_second_dining_hall/
    L10_hotel/
```

仓库中可以先保留目录说明。第二周本地已将群收集小程序中的现有样本复制到 `datasets/landmarks/`，但图片原图仍按 `.gitignore` 排除，不直接提交 Git；课程提交或算法重建时可使用本地目录、网盘或压缩包同步。

## 图片命名

格式：

```text
<landmarkCode>_<angle>_<light>_<index>.jpg
```

示例：

```text
L01_front_day_001.jpg
L01_side_cloudy_002.jpg
L04_far_day_001.jpg
```

推荐取值：

| 字段 | 说明 | 示例 |
| --- | --- | --- |
| `angle` | 拍摄角度 | `front`、`side`、`far`、`near`、`entrance` |
| `light` | 光照条件 | `day`、`cloudy`、`evening`、`backlight` |
| `index` | 三位序号 | `001`、`002` |

## 采集要求

- 每个地标不少于 20 张图片。
- 覆盖正面、侧面、远景、近景和典型入口或标志物。
- 尽量覆盖不同光照和距离。
- 删除严重模糊、主体过小、遮挡过多、重复度过高的图片。
- 每个地标至少选择 1 张代表图用于前端展示。

## 元数据

每个地标至少维护：

- 地标编号。
- 中文名称。
- 英文或拼音名称。
- 类型。
- 位置描述。
- 简介。
- 代表图片路径。
- 平面图静态标注位置。

## 图像库管理

- 地标样本图片按 `datasets/landmarks/` 下的 L01-L10 子目录管理，每个地标对应一个固定目录。
- 图片原图不提交到 Git，通过 `.gitignore` 排除，避免仓库体积过大；课程交付、算法重建或组内同步时，可使用本地目录、网盘或压缩包提供样本。
- 当前样本以 `.jpg` 图片为主，便于算法服务统一读取和特征提取；如后续补充其他格式，应先转换或在算法脚本中明确兼容规则。
- 每个地标选择 1 张最具代表性的图片作为展示图，并在数据库 `landmark` 表的 `cover_image_url` 字段中维护对应路径或图片内容。

## 样本质量控制

### 已识别风险

| 风险类型 | 风险描述 | 影响范围 | 严重程度 | 关联模块 |
| --- | --- | --- | --- | --- |
| 命名规范不一致 | 部分图片可能使用非推荐角度或光照取值，影响后续批量统计 | 样本整理、算法索引重建 | 中 | M2、M3 |
| 图片质量参差不齐 | 模糊、遮挡、主体过小或重复度过高的图片会降低特征稳定性 | 特征提取、检索排序 | 中 | M3 |
| 光照覆盖不均 | 若部分地标主要集中在单一光照条件，检索鲁棒性会下降 | 检索准确率 | 低 | M3 |
| 角度覆盖不均 | 缺少远景、近景或入口等关键视角时，容易影响同类建筑区分 | 地标识别、前端展示 | 低 | M2、M3、M4 |
| 代表图未统一确认 | 展示图与数据库 `cover_image_url` 不一致时，会影响详情页和地图标注效果 | 前端展示、数据库维护 | 中 | M2、M4 |

### 控制措施

| 风险类型 | 控制措施 | 责任模块 | 完成状态 |
| --- | --- | --- | --- |
| 命名规范不一致 | 对照命名格式批量检查文件名，必要时统一重命名并记录来源 | M2 | 进行中 |
| 图片质量参差不齐 | 人工剔除严重模糊、重复度高、主体过小或标签不确定的图片 | M2 | 待执行 |
| 光照覆盖不均 | 后续补采时优先补充阴天、傍晚、背光等弱覆盖条件 | M2 | 待执行 |
| 角度覆盖不均 | 按正面、侧面、远景、近景、入口等角度检查覆盖率 | M2 | 待执行 |
| 重复图片 | 算法侧可使用图像相似度或特征距离辅助识别重复样本 | M3 | 待执行 |
| 代表图未统一确认 | 与前端展示口径统一代表图选择标准，并同步数据库字段 | M2、M4 | 待执行 |

### 质量验收标准

| 指标 | 验收标准 |
| --- | --- |
| 图片数量 | 每个地标不少于 20 张有效样本 |
| 命名规范 | 文件名符合 `<landmarkCode>_<angle>_<light>_<index>.jpg` 或在整理清单中保留可追溯来源 |
| 角度覆盖 | 每个地标尽量覆盖正面、侧面、远景、近景和典型入口 |
| 光照覆盖 | 每个地标至少覆盖白天或阴天等常见拍摄条件，后续继续补充弱覆盖场景 |
| 图片质量 | 无严重模糊，主体清晰，遮挡和重复度控制在可接受范围内 |
| 可追溯性 | 样本数量、来源目录和后续补采重点记录在 `datasets/landmarks/sample_inventory.md` |
