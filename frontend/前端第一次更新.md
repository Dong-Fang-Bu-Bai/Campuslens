# CampusLens 前端开发更新说明文档

本说明文档针对 CampusLens 校园地标图像检索与导览系统的首次前端页面重构与交互优化进行技术性记录。本次更新旨在提升界面视觉秩序、优化长页面信息读取效率、规范组件交互逻辑，并消除单页应用（SPA）下的视图切换生硬感。

本次重构与更新工作严格遵循 Web 前端规范与现代软件工程标准，对以下技术模块进行了深度改动。

---

## 一、 UI 配色与视觉框架调整 (Visual Framework)

为了提高系统作为智能化 AI 图像检索终端的专业质感，全站视觉规范重塑为极简的暗色模式（Dark Mode）：
1. **主背景色重设**：将原有的浅色沙泥色背景替换为深暗曜石色（`#070913`），作为全局底层支撑，消除大片空旷区域的视觉发散感，并有效保护视力。
2. **卡片容器规范**：列表项、反馈面板及详情面板一律采用半透明深碳黑碳素材质色（`#0d121f`，带 `backdrop-filter: blur(...)` 亚克力透光质感），并使用 `1px` 精细的低对比度发光边界（`rgba(255, 255, 255, 0.03)`）进行几何勾勒，强化模块层级与空间深度。
3. **高亮强调色标准化**：强调色、图标与选中激活态完全向一线标准技术规范看齐，统一采用科技蔚蓝色（`#3b82f6`）与中性暗钢灰（`#475569`）作为核心指示，剔除原有的高饱和度翡翠绿和亮橙色。

## 二、 布局结构优化与信息密度重组 (Layout Restructuring)

针对宽屏设备下局部界面产生的布局单薄和空洞感，对 Results、Map、Feedback 三个核心视图的布局进行了如下深度优化：
1. **Results 视图：AI 算法指标面板 (Top Metrics Card)**：
   在 Results 卡片列表最上方，新增一排由三卡片组成的状态看板。第二周联调版本已将早期静态指标替换为接口返回的 `searchRecordId`、候选数量和检索状态，避免展示未经验证的准确率和样本统计。
2. **Map 视图：科技网格背景线 (Grid Map Background)**：
   为地图外部视口容器（`.map-viewport`）通过 CSS `linear-gradient` 配置了 20px x 20px 的精细直角坐标网格背景线。即使地图底图在百分比contain自适应模式下收缩，视口四周也有秩序井然的网格数据底色支撑，彻底解决地图边缘留白空洞问题。
3. **Feedback 视图：双栏控制面板自适应 (Dual-Column Layout)**：
   抛弃了单表单单栏的不平衡版面，重构为双栏黄金比例排版：
   * **左侧**：引入“最近检索历史流水日志看板”，实时展示文件名、检索置信度、流水时间及“已验证 / 需纠错”的状态状态，提高系统的业务厚度；
   * **右侧**：放置原有的“纠错反馈表单组件”。版面充实度显著提升。

## 三、 过渡动画与微交互系统实现 (Transitions & Micro-interactions)

为消除原本直接使用 `v-if` 进行物理闪烁替换的粗糙感，建立了平滑的 CSS 阻尼过渡与 Vue 过渡组件架构：
1. **Vue Transition 路由视图渐入淡滑过渡 (Fade and Slide)**：
   Results、Map、Feedback 页面在切换时被包裹于 `<Transition name="page-fade" mode="out-in">` 组件内。切换模块时，旧模块向上淡出，新模块从下方伴随三维阻尼垂直平移滑入，耗时 300ms，达成丝滑的页面切换手感。
2. **列表卡片级联延迟淡入动画 (Staggered Load Animation)**：
   Results 列表卡片重写了出现动效，利用 CSS 的 `animation-delay` 对 5 张卡片配置阶梯延迟。检索结果渲染时，卡片从上到下阶梯式地依次载入，视觉节奏极佳。
3. **Stripe 级侧栏状态指示蔚蓝滑块 (Active Sidebar Line)**：
   选中侧栏按钮时，按钮左侧会平滑地垂直拉伸出一根极细的蔚蓝高亮科技光线（`active-line`），并在 CSS `scaleY(1)` 的过渡中磁吸定位，菜单切换逻辑高雅低调。
4. **检索按钮扫描流光特效 (Loading Scan Line)**：
   在检索数据处理（Loading）期间，提交按钮启用 `is-loading` 状态，其表层自动加载一条横向平移无限循环的冷蓝渐变流光，用于指代后台算法的实时图像解析计算。

## 四、 核心可用性与交互逻辑修复 (UX & Usability Fixes)

1. **Results 面板右侧详情：60 FPS Sticky 粘性定位**：
   将结果详情说明卡片（`.detail-panel`）的定位方式变更为 `position: sticky; top: 24px; align-self: start;`。当用户向下划动 Results 列表查看更多结果时，右侧详情卡片将以 60 帧速度平滑跟随视口固定在屏幕右侧，无需反复回滚。
2. **高规格地标详情模态弹出组件 (Detail Modal Component)**：
   重构“详情”点击交互。点击卡片后，在屏幕正中调出独立的模态详情弹窗：
   * **实拍大图配网格技术参数**：为地标种子数据配置了高清晰度的经典校园实拍风景图（具有渐变遮罩），并配备“深度解析技术指标网格”；
   * **三种关闭逻辑**：支持右上角 Close 叉号关闭、点击外部遮罩层关闭，以及 **全局监听键盘 Escape 键一键退出**。弹出过程配备平滑的 `modal-scale-in` 物理回弹动效。
3. **SPA 全局历史导航状态机 (SPA Navigation History State Machine)**：
   在 Vue 顶层建立了一个 `viewHistory`（数组）和 `historyIndex`（整数指针）导航栈。拦截用户在 Results、Map、Feedback 之间的操作和检索跳转。自动根据当前历史记录状态计算顶栏 `<`（后退）和 `>`（前进）按钮的交互禁用（Disabled）状态，实现单页应用下的自由后退与前进。
4. **地标 Marker 绝对几何居中与无延迟对齐**：
   重置了 Marker 打点按钮的默认边距。通过将打点数字包裹于 `span` 中，并锁死 `line-height: 20px`（精确对应 Marker 24px 圆环扣减 4px 边框后的可视物理区域），彻底解决打点文字在不同浏览器中显示偏移的问题。同时，移除了 Marker 缩放和拖拽时的 `left`/`top` CSS 过渡动画，杜绝缩放地图时打点滞后产生的“漂移”感。
